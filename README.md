## Overview

`push_swap` is a 42 project that involves sorting a stack of integers using a limited set of operations. The goal is to sort the numbers in the smallest possible number of operations. This project provides an implementation of the `push_swap` algorithm.

The program takes a list of integers as arguments, pushes them onto a stack `a`, and then sorts them into ascending order using a second empty stack `b` and a predefined set of operations.

## Features

-   **Stack Manipulation:** Implements all required `push_swap` operations (swap, push, rotate, reverse rotate).
-   **Error Handling:** Robust error checking for invalid input (non-numeric arguments, duplicates, out-of-integer-range numbers).
-   **Sorting Algorithm:** An efficient algorithm designed to sort various sizes of integer lists with an optimized number of operations.
-   **Libft Integration:** Utilizes the `libft` library for common utility functions.

## How it Works (Implementation Details)

This implementation typically employs a strategy that involves:

1.  **Parsing and Validation:** Reads the input arguments, converts them to integers, and validates them for correctness (no duplicates, only numbers, within integer limits).
2.  **Stack Initialization:** Initializes stack `a` with the input numbers and an empty stack `b`.
3.  **Sorting Strategy:**
    *   For small numbers of elements (e.g., 2, 3, 4, 5), a specific, optimized sequence of operations is used.
    *   For larger numbers, a more complex algorithm is employed. A common approach involves:
        *   Moving elements from stack `a` to stack `b` strategically, often pushing the smallest or largest elements first, or using a "chunking" method.
        *   Calculating the "cost" of moving each element from `b` back to `a` to find the most efficient moves (e.g., minimizing rotations on both stacks).
        *   Performing rotations on both stacks simultaneously (`rr`, `rrr`) when beneficial to reduce the total number of operations.
        *   Finally, rotating stack `a` to place the smallest element at the top.

## Compilation

To compile the `push_swap` program, navigate to the root directory of the project and run `make`:

```bash
make
```

This will compile the `push_swap` executable and the `libft` library.

To clean up object files:

```bash
make clean
```

To clean up object files and the executable:

```bash
make fclean
```

To recompile the project:

```bash
make re
```

## Usage

The `push_swap` program takes a list of integers as command-line arguments.

```bash
./push_swap [list_of_integers]
```

**Example:**

```bash
./push_swap 2 1 3 6 5 8
```

The program will output the sequence of operations required to sort the numbers to standard output.

### Using the Checker

The `checker_linux` executable (provided in the project) can be used to verify if the sequence of operations generated by `push_swap` correctly sorts the given numbers. You can pipe the output of `push_swap` directly to the checker:

```bash
./push_swap 2 1 3 6 5 8 | ./checker_linux 2 1 3 6 5 8
```

The checker will output `OK` if the stack is sorted, or `KO` if it's not, or `Error` if the operations are invalid.
